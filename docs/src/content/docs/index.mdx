CSP is built for the folders where agents and humans share working memory:
notes, plans, prompts, task state, and project context that needs to follow
you between a CLI, a desktop app, an editor plugin, or a remote machine.

Context stays as normal files on disk, so any app or agent reads and edits it
directly. CSP watches for changes, syncs them in real time, keeps a
stock-git-compatible history, and lets you recover the folder from an earlier
point in time — without ever writing conflict markers into your files.

## One core, thin bindings

Protocol behavior lives in Rust once. Every surface — CLI, SDK, plugin,
desktop — calls into the same engine, so behavior parity is structural, not
hand-maintained.
